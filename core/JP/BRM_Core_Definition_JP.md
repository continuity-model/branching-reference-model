# BRM Core Definition — 日本語版

## Branching Reference Model

### 核心的洞察
記憶の目的は保持ではない。
記憶は共有された参照空間を確立するために存在する。

人間とAIの協働が破綻する原因は知能の不足ではない。
人間とAIが同じ参照点に立っていないことにある。

ハルシネーション、文脈ドリフト、誤解、認知疲労はすべて参照の不一致によって起こる。

### 根本問題
従来のLLM対話は一本道文脈モデルに基づいている。

過去情報は一つに統合され、直近が優先され、推論が事実に混入し、決定が歪む。
その結果、参照は不安定になる。

### BRMの解決
BRMは文脈の強制統合を放棄し、分岐型参照ナビゲーションを導入する。

情報は分岐として保存され、ノードとして隔離され、リンクで接続される。
AIは記憶するのではなくナビゲートする。

### BRMとは
BRMは記憶DBでもプロンプト技法でもない。
人間とAIの参照を一致させるためのプロトコルである。

### 最終定義
BRMは記憶構造ではない。
人間とAIが同じ参照空間に立つためのプロトコルである。

記憶の目的は保持ではない。
参照可能な分岐を提供することである。
