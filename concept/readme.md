# Concept — BRM Comparisons

This section explains how the **Branching Reference Model (BRM)** relates to other common AI approaches.

These pages are not theoretical discussions.

They show **where BRM fits**, and what problem each approach is actually solving.

---

## What This Covers

Most AI workflows rely on one of the following:

- default conversation (no structure)
- prompting
- retrieval (RAG)
- agents

Each solves a different problem.

BRM focuses on something else:

👉 **reasoning stability and reference control**

## Key Insight

BRM is not a replacement for other approaches.

It operates at a different layer.

- Prompt → behavior control  
- RAG → knowledge layer  
- Agent → execution layer  
- BRM → **structure layer**

---

## One Line

RAG adds knowledge.  
Agents add capability.  
BRM adds stability.
